# Quick Start Guide

این راهنما دقیق و قدم‌به‌قدم توضیح می‌دهد که روی لپ‌تاپ چطور این پروژه را از صفر راه‌اندازی کنید.

## 1) پیش‌نیازها

قبل از شروع، مطمئن شوید این ابزارها روی سیستم شما نصب هستند:

- Node.js 20 یا بالاتر
- pnpm
- Docker Desktop (یا Docker Engine) برای اجرای PostgreSQL و Redis
- Git
- یک ترمینال مانند Terminal، iTerm یا VS Code Terminal

### چک کردن نصب‌ها

در ترمینال این دستورات را اجرا کنید:

```bash
node -v
npm -v
pnpm -v
docker --version
git --version
```

اگر هر کدام نصب نبود، اول آنها را نصب کنید.

### اگر pnpm نصب نیست

```bash
npm install -g pnpm
```

---

## 2) کلون کردن پروژه

```bash
git clone <آدرس-ریپوزیتوری>
cd castaminofen-FULLStack-starter
```

اگر پروژه را از قبل دارید، فقط وارد پوشه شوید:

```bash
cd castaminofen-FULLStack-starter
```

---

## 3) نصب依ست‌ها

در root پروژه این دستور را اجرا کنید:

```bash
pnpm install
```

این کار ممکن است چند دقیقه طول بکشد. اگر با خطای وابستگی مواجه شدید، دوباره اجرا کنید:

```bash
pnpm install --frozen-lockfile
```

---

## 4) راه‌اندازی محیط متغیرها

یک فایل محیطی از نمونه بسازید:

```bash
cp .env.example .env
```

سپس فایل .env را باز کنید و اگر لازم بود این مقادیر را بررسی کنید:

```env
NODE_ENV=development
PORT=3000
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/castaminofen
REDIS_URL=redis://localhost:6379
```

> اگر روی سیستم شما PostgreSQL و Redis از قبل نصب هستند، این مقادیر معمولاً درست‌اند. اما اگر از Docker استفاده می‌کنید، این تنظیمات مناسب‌اند.

---

## 5) راه‌اندازی دیتابیس و Redis با Docker

در ریشه پروژه:

```bash
docker compose up -d postgres redis
```

اگر پروژه از قبل در حال اجرا است، می‌توانید با این دستور بررسی کنید:

```bash
docker compose ps
```

### تست اینکه سرویس‌ها فعال‌اند

```bash
docker ps
```

اگر PostgreSQL درست اجرا شد، باید بتوانید به پورت 5432 وصل شوید.

---

## 6) راه‌اندازی دیتابیس پروژه

این پروژه از Prisma استفاده می‌کند. برای آماده‌سازی دیتابیس، این دستور را اجرا کنید:

```bash
pnpm prisma migrate dev
```

اگر هنوز Seed وجود دارد، می‌توانید این دستور را هم اجرا کنید:

```bash
pnpm prisma db seed
```

اگر در پروژه Seed تنظیم نشده باشد، این مرحله را رد کنید و فقط به مرحله بعد بروید.

---

## 7) اجرای پروژه

برای اجرای همه سرویس‌ها به‌صورت همزمان:

```bash
pnpm dev
```

این دستور معمولاً پروژه‌های مختلف را هم‌زمان راه‌اندازی می‌کند.

### سرویس‌های پیش‌فرض

- API معمولاً روی پورت 3000 یا پورت تنظیم‌شده در .env اجرا می‌شود
- Web App معمولاً روی پورت دیگری در دسترس است

برای بررسی خروجی ترمینال، به‌دقت ببینید که کدام پورت‌ها فعال شده‌اند.

---

## 8) باز کردن پروژه در مرورگر

پس از اجرای موفق، این آدرس‌ها را در مرورگر باز کنید:

- http://localhost:3000
- یا اگر وب اپ روی پورت دیگری اجرا شد، آن پورت را باز کنید

اگر در نسخه‌های بعدی پروژه، چند سرویس جداگانه داشته باشید، خروجی ترمینال نشان می‌دهد کدام آدرس‌ها فعال‌اند.

---

## 9) اگر بخواهید فقط یک بخش خاص را اجرا کنید

### اجرای API

```bash
pnpm --filter @castaminofen/app-api dev
```

### اجرای وب

```bash
pnpm --filter @castaminofen/app-web dev
```

---

## 10) تست و بررسی کد

برای تست پروژه:

```bash
pnpm test
```

برای lint:

```bash
pnpm lint
```

برای ساخت نسخه production:

```bash
pnpm build
```

---

## 11) اگر با خطا مواجه شدید

### خطای Docker

اگر Docker کار نمی‌کند:

```bash
docker compose down
docker compose up -d postgres redis
```

### خطای pnpm install

```bash
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

### خطای دیتابیس

```bash
docker compose restart postgres
```

### خطای Port در حال استفاده است

اگر یک پورت مشغول است، یکی از این کارها را انجام دهید:

- پورت را تغییر دهید
- سرویس قبلی را متوقف کنید
- از پورت دیگری استفاده کنید

---

## 12) ساختار کوتاه پروژه

برای اینکه راحت‌تر در پروژه حرکت کنید، این ساختار مهم است:

- apps/api: بک‌اند و API
- apps/web: فرانت‌اند وب
- packages: ماژول‌ها و کتابخانه‌های مشترک
- prisma: مدل‌ها و migrations
- docker-compose.yml: راه‌اندازی سرویس‌های محلی

---

## 13) پیشنهاد عملی برای شروع

اگر بخواهید خیلی سریع وارد پروژه شوید، این ترتیب را دنبال کنید:

1. pnpm install
2. cp .env.example .env
3. docker compose up -d postgres redis
4. pnpm dev

---

## 14) نکته مهم

در این پروژه، برای اجرای کامل، ممکن است بسته به نسخه‌ی محیط شما، بعضی سرویس‌ها یا متغیرهای محیطی بیشتر از این‌ها نیاز داشته باشند. اگر بعد از اجرای دستورها هنوز چیزی کار نکرد، خروجی ترمینال را بخوانید و بر اساس خطا، مرحله‌ی مربوطه را اصلاح کنید.

اگر بخواهید، در مرحله بعد می‌توانم برایتان یک نسخه‌ی even more professional از همین فایل آماده کنم که شامل:

- راه‌اندازی روی Windows
- راه‌اندازی روی macOS
- رفع خطاهای رایج
- تفاوت بین Development و Production
- توضیح هر دستور به‌صورت فارسی دقیق‌تر
